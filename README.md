# Coworking Booking Service API 🏢

RESTful API сервис для бронирования рабочих мест в коворкингах. Реализовано на **Django Rest Framework**.

## ✨ Основные функции
- **Auth**: Регистрация и авторизация (Bearer Token).
- **Coworkings**: Управление площадками (только для админов).
- **Desks**: Управление рабочими местами и фильтрация по площадкам.
- **Bookings**: Бронирование мест на конкретные слоты (morning, afternoon, full_day) с защитой от дубликатов.
- **Permissions**: Разграничение прав (Гость, Клиент, Администратор).

## 🚀 Стек технологий
- Python 3.13+
- Django 5.x / 6.x
- Django Rest Framework
- SQLite (по умолчанию) / PostgreSQL

## 🛠 Установка и запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com
   cd название-проекта
   ```

2. **Создайте и активируйте виртуальное окружение:**
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # Для Linux/macOS
   .venv\Scripts\activate     # Для Windows
   ```

3. **Установите зависимости:**
   ```bash
   pip install django djangorestframework
   ```

4. **Примените миграции и создайте админа:**
   ```bash
   python manage.py migrate
   python manage.py createsuperuser
   ```

5. **Запустите сервер:**
   ```bash
   python manage.py runserver
   ```

## 📖 Примеры эндпоинтов

### Аутентификация
- `POST /auth/register/` — регистрация нового пользователя.
- `POST /auth/login/` — получение токена доступа.

### Коворкинги и места
- `GET /api/coworkings/` — список всех площадок.
- `GET /api/desks/?coworking_id={id}` — рабочие места в конкретном коворкинге.

### Бронирование
- `POST /api/bookings/` — создание брони (проверка на занятость слота).
- `POST /api/bookings/{id}/cancel/` — отмена активной брони.

## 🔒 Права доступа (Permissions)

| Метод | Гость | Клиент | Администратор |
| :--- | :---: | :---: | :---: |
| Просмотр площадок | ✅ | ✅ | ✅ |
| Создание площадок | ❌ | ❌ | ✅ |
| Создание брони | ❌ | ✅ | ❌ |
| Отмена своей брони | ❌ | ✅ | ✅ |
| Просмотр всех броней| ❌ | ❌ | ✅ |

## ⚠️ Валидация
Система автоматически возвращает `409 Conflict`, если выбранное место на указанную дату и временной слот уже занято другим пользователем.
